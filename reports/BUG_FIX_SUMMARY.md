## Bugs Fixed

### Bug 1: Weight Adjustment Logic Was BACKWARDS

**Location:** `src/agents/atlas_agent.py` - `Agent.update_weight()` method

**Problem:**
The original logic compared `recent_brier` to `overall_brier` to decide weight adjustment:
- If recent < overall: increase weight (improving)
- If recent > overall: decrease weight (worsening)

This is FLAWED because:
1. An agent with consistently BAD performance but slightly improving would get INCREASED weight
2. An agent with consistently GOOD performance but slight dip would get DECREASED weight
3. **This explains why MACD Trend (58.9% win rate - BEST) had weight 0.27 while others had 5.0**

**Fix:**
```python
# Now uses absolute performance (Brier score)
brier_threshold = 0.22  # Good performance threshold

if overall_brier < brier_threshold:
    # Good performer - increase weight
    performance_bonus = (brier_threshold - overall_brier) / brier_threshold
    adjustment = alpha * (1 + performance_bonus)
    self.weight = min(self.weight * (1 + adjustment), self.max_weight)
else:
    # Poor performer - decrease weight
    performance_penalty = (overall_brier - brier_threshold) / brier_threshold
    adjustment = alpha * (1 + min(performance_penalty, 1.0))
    self.weight = max(self.weight * (1 - adjustment), self.min_weight)
```

### Bug 2: Regime Filtering Excluded 4 Agents

**Location:** `src/agents/atlas_agent.py` - `AgentTeam.DEFAULT_AGENT_CONFIGS` and `predict()` method

**Problem:**
4 agents were configured with specific regimes that rarely matched:
- `momentum_hawk`: "TRENDING_UP" - never matched
- `volume_whale`: "BREAKOUT" - never matched
- `volatility_watcher`: "VOLATILE" - never matched
- `sentiment_reader`: "REVERSAL" - never matched

The detected regime was usually "RANGING", so these agents NEVER participated in predictions!

**Fix:**
1. Changed most agents to "ALL" regime (always active)
2. Updated filtering logic to include ALL agents (no filtering)
3. Added regime-matching bonus: agents whose regime matches get 1.5x weight bonus

### Bug 3: UP Direction Bias

**Location:** Multiple places in `src/agents/atlas_agent.py`

**Problem:**
- 70% more UP predictions than DOWN (24 vs 10)
- Prompts had subtle bullish bias
- Only 4 agents voting (the ALL-regime ones) created imbalanced signals

**Fix:**
1. Updated agent system prompts to explicitly require balanced predictions:
   ```
   IMPORTANT: You must be EQUALLY willing to predict UP or DOWN based on the data.
   - Do NOT have a bullish bias
   - Do NOT have a bearish bias
   - Let the data dictate the direction
   ```

2. Changed direction thresholds from 0.53/0.47 to 0.50 (simple majority)

3. All 8 agents now participate, providing more diverse signals

### Bug 4: Narrow Probability Range

**Location:** `src/agents/atlas_agent.py` - `_aggregate_predictions()` method

**Problem:**
All predictions clustered around 50% (0.40-0.56), making it hard to identify high-confidence trades.

**Fix:**
1. Widened direction decision threshold from 0.53/0.47 to 0.50
2. Improved confidence calculation to reward distance from 50%
3. Added agreement bonus when most agents agree
